# Engineering Portfolio Website Specification

## Project Vision

Develop a professional engineering portfolio website that presents my work as if it were official semiconductor documentation rather than a traditional personal portfolio.

The website should resemble a modern digital engineering datasheet while remaining fully responsive, accessible, and easy to navigate.

The final product should communicate precision, professionalism, engineering rigor, and attention to detail.

---

# Design Philosophy

The website should feel like:

- Texas Instruments Datasheets
- Analog Devices Application Notes
- IEEE Publications
- Laboratory Documentation
- Engineering Manuals
- Instrumentation Documentation

Avoid:

- SaaS landing pages
- Startup websites
- Glassmorphism
- Tailwind templates
- Framer aesthetics
- Generic portfolio layouts

# Implementation Rules

Write production-quality code.

Comment major sections.

Use semantic HTML.

Prefer reusable CSS.

Avoid unnecessary JavaScript.

Performance is an engineering requirement.

Maintain the engineering documentation metaphor throughout.

Whenever making a design decision ask:

"Would this feel believable inside official documentation from Texas Instruments, Analog Devices, Keysight, Tektronix, Fluke, or IEEE?"

If not, redesign it.
