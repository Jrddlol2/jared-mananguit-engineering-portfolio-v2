1

Overview

A full custom 8-bit CPU based on the SAP-2 (Simple-As-Possible) architecture, designed and verified in Verilog from the ground up rather than assembled from a black-box IP core. The design spans a complete datapath, a hand-built control unit, and a 42-instruction ISA, with every instruction's fetch-decode-execute cycle verified against ModelSim waveforms.

2

Instruction Cycle

Each instruction executes across a sequence of T-states generated by a ring-counter sequencer: fetch states retrieve the opcode from memory into the instruction register, decode states resolve it into a one-hot active-instruction signal, and execute states assert exactly the register loads, bus enables, and ALU controls that instruction requires for that T-state. Multi-byte and multi-cycle instructions (such as CALL/RET and memory-reference addressing) extend the T-state sequence accordingly before the cycle resets to fetch the next instruction.

3

Datapath

An 8-bit data bus (the W-Bus) and a 16-bit address bus connect every functional register, with tri-state bus arbitration ensuring only one register drives the W-Bus per T-state. Program memory is 256 bytes (128 B ROM / 128 B RAM), with the initial program image loaded via Verilog's $readmemh from a hex file.

5

Control Logic

The control unit is built from three parts: a ring counter acting as the T-state sequencer; an instruction decoder that turns the fetched opcode into a 32-bit one-hot active-instruction signal; and a combinational control matrix that ANDs the active-instruction signal against the current T-state to generate every register load/enable line and ALU control signal directly — no microcode ROM. This made the timing of every control signal explicit and traceable back to a specific instruction/T-state combination.

Implementation

  • 42-instruction ISA spanning memory-reference, register, jump/call, logical, and special (conditional jump/call/return-if-minus) categories, including subroutine CALL/RET via a dedicated return-address register.
  • ALU supporting add/subtract (two's-complement), AND/OR/XOR, increment/decrement, and left/right rotate, with sign and zero flags feeding conditional control flow.
  • 256-byte memory (128 B ROM / 128 B RAM) with the program image loaded via $readmemh.
  • Tri-state bus arbitration across every register sharing the 8-bit W-Bus.

Each instruction was verified against a dedicated ModelSim testbench before the full integrated fetch-decode-execute simulation was run, so failures could be isolated to a specific instruction rather than debugged from a whole-program waveform.

9

Lessons Learned

Building the control matrix combinationally, rather than as a microcode ROM, forced a much more precise understanding of exactly which control line needed to assert on exactly which T-state for every instruction — a shortcut microcode approach would have hidden bugs that per-T-state combinational logic surfaced immediately in simulation. Verifying instructions individually before integrated testing also caught several control-matrix timing bugs far earlier than a whole-program-first approach would have.
