Executive Summary

A self-hosted network-attached storage platform, designed as a full systems-engineering exercise: hardware selection, Linux installation and hardening, network configuration, redundant storage layout, access security, and an automated backup strategy — rather than deploying a commercial NAS appliance as-is. The goal is centralized, resilient file storage for personal and project use, administered entirely from the command line.

2

Problem Statement

Project files, datasets, and media were scattered across individual machines with no redundancy and no single point of access from the local network. A single-disk backup is not fault-tolerant, and cloud storage alone does not provide the low-latency LAN access needed for large engineering files (KiCad projects, LTspice runs, simulation data).

6

Linux Operating System

A Debian-based Linux distribution is planned as the host OS, administered entirely via SSH/CLI — deliberately avoiding a GUI-first NAS appliance OS to keep the full stack transparent and directly configurable (systemd units, filesystem tools, and firewall rules under direct control).

7

Network Configuration

Static IP addressing on the LAN segment, SMB for Windows/macOS client access, and NFS for Linux client access. Reverse-proxied access for any web-facing management dashboard, kept off the public internet behind the home router's firewall.

8

Storage Configuration

Redundant array (RAID 1 or RAID-Z1 equivalent, depending on final disk count) to tolerate a single-disk failure without data loss, with periodic scrub/integrity checks scheduled via cron/systemd timers.

9

Security Considerations

  • SSH key-based authentication only; password authentication disabled.
  • Per-user share permissions rather than a single shared-everything volume.
  • Host firewall restricting SMB/NFS access to the local subnet only.
  • Regular OS/package updates on a defined patch cadence.
10

Backup Strategy

Automated daily incremental and weekly full backups to a separate physical backup target, following a 3-2-1 pattern (three copies, two media types, one off-site/offline) as far as available hardware permits.

Engineering Decisions

Chose a hand-configured Debian base over a turnkey NAS distribution to maximize transparency and control over every layer of the stack — accepting a steeper initial setup cost in exchange for full visibility into the network, storage, and security configuration, consistent with the systems-level approach applied throughout this document.
